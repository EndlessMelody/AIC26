# Data Specification

The actual 2026 data spec will be published by the organizers shortly before the preliminary round.
This document captures **expected** structure based on 2022-2025 editions so we can build the pipeline ahead of time.

## Expected Volume
- ~300 hours of Vietnamese news video, released in 3 batches:
  - Batch 1 (~100h): for familiarisation.
  - Batch 2 (+100h): used for the preliminary round.
  - Batch 3 (+100h): added for the final round.

## File Layout Provided by Organizers (typical)
```
release/
├── videos/                       # MP4 files
│   ├── L01_V001.mp4
│   ├── L01_V002.mp4
│   └── ...
├── keyframes/                    # Pre-extracted JPGs
│   ├── L01_V001/
│   │   ├── 0001.jpg
│   │   ├── 0002.jpg
│   │   └── ...
│   └── ...
├── clip-features-vit-b16/        # Pre-computed CLIP features
│   ├── L01_V001.npy              # shape [N_keyframes, 512]
│   └── ...
├── objects/                      # Faster R-CNN object detections
│   ├── L01_V001/
│   │   ├── 0001.json             # 100 objects from 600 Open Images classes
│   │   └── ...
│   └── ...
├── map-keyframes/                # Frame -> timestamp mapping
│   ├── L01_V001.csv              # columns: n, pts_time, fps, frame_idx
│   └── ...
└── metadata/                     # Optional descriptions
    └── L01_V001.json
```

## Key IDs and Conventions
- `video_id`: `Lxx_Vyyy` where `xx` is the batch and `yyy` is the video index.
- `keyframe_id`: zero-padded sequence number within the video, e.g. `0001`.
- `frame_idx`: the absolute frame number in the source video (for submission).
- `timestamp_ms`: keyframe time in milliseconds from the video start.

## Local Working Layout (under `data/`)
```
data/
├── raw/                          # Symlink or copy of BTC videos
├── keyframes/                    # Either BTC keyframes or our own re-extraction
├── metadata/
│   ├── ocr.parquet               # OCR text per keyframe (Vietnamese)
│   ├── asr.parquet               # ASR transcripts per video segment
│   ├── objects.parquet           # Object detections (BTC + our own)
│   ├── descriptions.parquet      # Captions generated by BLIP-2 / LLaVA
│   └── manifest.json
├── index/
│   ├── clip_vit_l14/
│   │   ├── embeddings.npy
│   │   ├── ids.json
│   │   └── hnsw.faiss
│   ├── siglip_so400m/...
│   └── bm25/...
├── queries/
│   ├── sample.json               # Hand-crafted queries for local eval
│   └── official/                 # Official preliminary / final queries
└── models/                       # HuggingFace cache (set HF_HOME here)
```

## Metadata Schemas

### `ocr.parquet`
| column | type | notes |
|---|---|---|
| video_id | string | |
| keyframe_id | string | zero-padded |
| text | string | concatenated OCR text in reading order |
| boxes | list<list<float>> | bounding boxes (x, y, w, h) normalized |
| confidence | float | average confidence |

### `asr.parquet`
| column | type | notes |
|---|---|---|
| video_id | string | |
| start_ms | int64 | segment start |
| end_ms | int64 | segment end |
| text | string | Vietnamese transcript |
| confidence | float | |

### `objects.parquet`
| column | type | notes |
|---|---|---|
| video_id | string | |
| keyframe_id | string | |
| labels | list<string> | object class names |
| scores | list<float> | detection scores |
| boxes | list<list<float>> | normalized boxes |

### `manifest.json`
Document the provenance of every artefact:
```json
{
  "clip_vit_l14": {
    "model": "open_clip:ViT-L-14/laion2b_s32b_b82k",
    "dim": 768,
    "normalized": true,
    "created_at": "2026-04-01T12:00:00Z",
    "num_items": 1234567
  }
}
```

## Storage Sizing (rough estimate)
| Artefact | Size for ~300h |
|---|---|
| Raw MP4 | 200 - 400 GB |
| Keyframes (JPG) | 50 - 100 GB |
| CLIP embeddings ViT-L (fp16) | ~2 - 5 GB |
| FAISS HNSW index | ~3 - 8 GB |
| OCR + ASR parquet | ~1 - 3 GB |

Plan for **at least 1 TB** of fast SSD storage on the working machine.
